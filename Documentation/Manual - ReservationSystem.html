Instructions

    The time it takes to fill up at a pump is 30ms, and since the station has two pumps, each pump can be used simultaneously.  There is no order for trying to fill up so the distribution will be on a first come first served basis. 

When the simulation starts, all the cars should be ready to start (their threads have started and they are waiting to go).  Once started each car should try to fill up by picking a free pump at the station, marking that pump as in-use, fill for 30 ms, free up the pump and then try to fill up again.

When the simulation is over you need to print out the fill up count for each car and pump.  The total number of fill-ups per pump should be approximately the same (+/- 5) and should be close to the length of the test divided by 30 ms.  The distribution of the fill ups for all of the cars should be fair, for instance, no cars should have zero fill ups.

The code for this final station.cpp is designed with a number of joinable threads that run each car.  The threads continue working until the test is over.  Your job is to modify the code to enable the gas station simulation to run for a hardcoded time of 30 seconds, with 10 cars and 2 pumps and then shutdown so the results can be spit out.

For further testing you can change the command line parameters: cars, pumps, and time for test.
